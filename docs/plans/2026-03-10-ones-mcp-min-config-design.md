# ONES MCP 最小配置化设计文档

**日期：** 2026-03-10  
**目标：** 将当前 MCP 改为“仅配置登录信息即可使用”，并支持从上下文中的完整 URL 或 `#需求号` 获取关联文档正文。

## 1. 设计范围

本次改造覆盖以下行为：

1. 环境变量最小化：仅保留 ONES 地址与登录凭据（+可选运行参数）。
2. `get_doc` 入参统一为 `ref`，支持完整 URL 与 `#12345`。
3. `#12345` 语义固定为“返回该需求号关联文档中最新的一篇正文”。
4. 自动探测 ONES 接口路径，不要求用户配置接口路径。
5. 提供明确错误码与可理解错误信息。

## 2. 配置模型

### 必填配置

- `ONES_BASE_URL`
- `ONES_USERNAME`
- `ONES_PASSWORD`

### 可选配置

- `ONES_TIMEOUT_MS`（默认 15000）
- `ONES_MAX_CONTENT_CHARS`（默认 20000）

### 非目标

- 不再支持用户显式配置 `ONES_LOGIN_PATH`、`ONES_SEARCH_PATH`、`ONES_DOC_PATH_TEMPLATE`。
- 不做向后兼容分支（老变量直接移除）。

## 3. 工具接口设计

### `search_docs`

- 入参：
  - `query: string`（必填）
  - `limit?: number`（默认 5，1-20）
- 行为：
  - 使用自动探测得到的“搜索接口”执行检索；
  - 返回标准化结果：`id/title/updated_at`。

### `get_doc`

- 入参：
  - `ref: string`（必填）
- 行为：
  - 若是完整 URL：提取文档标识后直接抓正文；
  - 若匹配 `#数字`：先查需求，再取关联文档，按最新更新时间选一篇抓正文；
  - 返回：`id/title/content/updated_at`（content 已清洗和截断）。

## 4. `ref` 解析规则

按以下顺序解析：

1. 完整 URL（`http://` 或 `https://`）
2. 需求号（`^#\\d+$`）
3. 其他格式：`INVALID_DOC_REF`

URL 解析时需要校验 host 与 `ONES_BASE_URL` 一致，防止误抓外部地址。

## 5. `#需求号` 关联文档策略

对于 `#12345`：

1. 通过“需求详情接口”获取该需求的关联文档列表；
2. 过滤无效文档条目（缺 ID）；
3. 优先按 `updated_at` 降序；
4. 若 `updated_at` 缺失，则回退 `created_at`；
5. 取最新一篇并抓正文；
6. 若无关联文档：返回 `NO_LINKED_DOC`。

## 6. 自动探测策略

### 探测时机

- 启动后惰性探测（首次调用时进行）并缓存结果。

### 探测目标

1. 登录接口路径（候选列表）
2. 搜索接口路径（候选列表）
3. 文档详情接口模板（候选模板）
4. 需求详情/关联文档接口路径（候选列表）

### 命中条件

- 登录：`2xx + Set-Cookie`
- 搜索：`2xx + 可解析列表结构`
- 文档/需求：`2xx + 可解析目标结构`

候选全部失败返回 `DISCOVERY_FAILED`。

## 7. 会话管理与重试

1. 复用现有 Cookie 缓存；
2. 请求返回 401 时：
  - 清空会话；
  - 自动重新登录；
  - 原请求仅重试一次；
3. 二次失败不再重试，返回 `AUTH_FAILED` 或 `UPSTREAM_ERROR`。

## 8. 错误码设计

- `INVALID_DOC_REF`：`ref` 不是完整 URL 也不是 `#数字`
- `NO_LINKED_DOC`：需求未关联文档
- `DISCOVERY_FAILED`：候选接口均不可用
- `AUTH_FAILED`：登录失败/会话恢复失败
- `NOT_FOUND`：目标不存在
- `UPSTREAM_ERROR`：其他上游异常

## 9. 测试与验收

### 单元测试

1. `ref` 解析（URL、`#12345`、非法输入）
2. `#12345` 关联文档排序（`updated_at`、`created_at` 回退）
3. 自动探测命中/失败
4. 401 重登一次重试机制

### 集成测试（Mock ONES）

1. `search_docs` 全链路
2. `get_doc(URL)` 全链路
3. `get_doc(#12345)` 全链路（含关联文档排序）

### 手工验收

仅设置三项登录配置即可完成：

1. 关键词搜索
2. URL 直取文档
3. `#需求号` 取关联文档最新正文

## 10. 风险与约束

1. 不同 ONES 租户接口差异会影响探测成功率，需要可维护的候选列表。
2. 关联文档字段可能因实例不同存在命名差异，需要兼容映射。
3. 自动探测需控制超时与重试，避免首请求等待过长。
